# Network Modes Overview

## Online Deployment

**Characteristics:**
- Internet-connected
- Can access cloud APIs
- Automatic updates
- Remote access possible

**Use Cases:**
- Public APIs
- Cloud hybrid apps
- SaaS products

## Offline Deployment

**Characteristics:**
- Air-gapped (no internet)
- All dependencies bundled
- Manual updates
- Maximum security

**Use Cases:**
- Government/military
- Healthcare
- Financial systems
- Air-gapped research labs

## Private Network Deployment

**Characteristics:**
- Internal network only
- VPN or firewall protected
- Controlled access
- Limited external connectivity

**Use Cases:**
- Enterprise apps
- Internal tools
- Research institutions

## Comparison

| Aspect | Online | Offline | Private |
|--------|--------|---------|---------|
| Internet | Yes | No | Limited |
| Security | Medium | Maximum | High |
| Updates | Auto | Manual | Controlled |
| Cost | Low | High | Medium |
| Complexity | Low | High | Medium |

## Hybrid Approaches

### Online-Offline Hybrid
- Primary: Offline operation
- Secondary: Sync when online

### Private-Public Hybrid
- Internal: Private network
- External: Selective access
