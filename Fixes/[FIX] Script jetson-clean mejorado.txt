[FIX] Script jetson-clean mejorado

¡Tienes toda la razón! El script original falló estrepitosamente porque está intentando apagar contenedores con nombres fijos (`vllm-openclaw`, `llama-openclaw`) que **no coinciden con los nombres reales** de lo que tienes corriendo en tu Jetson.

### 🔍 El Diagnóstico de tu Terminal:

1. **Los culpables del consumo:** Tienes dos contenedores activos llamados **`vllm`** y **`qwen35-35b`**. El segundo se está tragando **36.57 GiB** de tu RAM. El script ni los tocó porque no sabía que existían.
2. **Falsos positivos:** El script dijo `"✓ No había contenedores LLM"` cuando en realidad `docker stop` imprimió `vllm-openclaw` indicando que lo intentó apagar, pero el operador `||` se ejecutó de forma confusa.
3. **Contenedores fantasma:** Tienes un contenedor `vllm` que se levantó hace solo 42 segundos. ¿Por qué? Porque al apagar `open-webui` (o por algún script externo), se disparó otro proceso.

---

### 🛠️ La Solución: Un script inteligente y dinámico

En lugar de adivinar los nombres de tus contenedores, vamos a modificar el script para que **detecte automáticamente cualquier contenedor que use vLLM o Llama.cpp** y lo apague, además de limpiar Ollama y la memoria caché de forma efectiva.

Vuelve a crear el script ejecutando este bloque en tu terminal:

```bash
cat > ~/scripts/jetson-clean.sh << 'EOF'
#!/bin/bash

mostrar_diagnostico() {
    echo "  [1] Memoria del Sistema:"
    free -h | awk 'NR==1 || NR==2' | sed 's/^/      /'
    
    echo "  [2] Contenedores Activos:"
    docker ps --format "table {{.Names}}\t{{.Status}}\t{{.RunningFor}}" | sed 's/^/      /'
    
    echo "  [3] Consumo de RAM por Contenedor:"
    docker stats --no-stream --format "table {{.Name}}\t{{.MemUsage}}\t{{.MemPerc}}" | sed 's/^/      /'
    
    echo "  [4] Modelos Ollama en GPU:"
    if systemctl is-active --quiet ollama; then
        ollama ps 2>/dev/null | sed 's/^/      /'
    else
        echo "      Ollama → offline (Servicio inactivo)"
    fi
    
    echo "  [5] Endpoints de Inferencia Activos:"
    # vLLM
    curl -s http://localhost:8000/v1/models 2>/dev/null | \
      python3 -c "import sys,json; [print('      vLLM:8000 →', m['id']) for m in json.load(sys.stdin)['data']]" \
      2>/dev/null || echo "      vLLM:8000 → offline"
    
    # Llama.cpp
    curl -s http://localhost:8080/v1/models 2>/dev/null | \
      python3 -c "import sys,json; [print('      llama.cpp:8080 →', m['id']) for m in json.load(sys.stdin)['data']]" \
      2>/dev/null || echo "      llama.cpp:8080 → offline"
}

echo "╔══════════════════════════════════════════════════════════════════════════╗"
echo "║                   ESTADO INICIAL (DIAGNÓSTICO)                          ║"
echo "╚══════════════════════════════════════════════════════════════════════════╝"
mostrar_diagnostico

echo ""
echo "╔══════════════════════════════════════════════════════════════════════════╗"
echo "║                       EJECUTANDO LIMPIEZA                                ║"
echo "╚══════════════════════════════════════════════════════════════════════════╝"
echo "→ Detectando e interactuando con inferencias activas..."

# 1. Obtener nombres de contenedores sospechosos
CONTAINERS=$(docker ps --format '{{.Names}}' | grep -E "vllm|llama|qwen|open-webui|openclaw")

if [ -n "$CONTAINERS" ]; then
    echo "  ⚠️ Eliminando contenedores para evitar autoregeneración..."
    echo "$CONTAINERS" | sed 's/^/    - /'
    docker rm -f $CONTAINERS >/dev/null 2>&1
    echo "  ✓ Contenedores eliminados."
else
    echo "  ✓ No se detectaron contenedores de IA corriendo."
fi

# 2. Kill a procesos rebeldes en el sistema host
echo "→ Limpiando procesos huérfanos de vLLM/Python en el Host..."
sudo pkill -f vllm 2>/dev/null
sudo pkill -f qwen 2>/dev/null
echo "  ✓ Procesos mitigados."

# 3. Descargar modelos de Ollama de la GPU (si aplica) y apagar servicio
if systemctl is-active --quiet ollama; then
    echo "→ Descargando modelos Ollama de GPU..."
    for m in $(ollama ps 2>/dev/null | tail -n +2 | awk '{print $1}'); do
      curl -s http://localhost:11434/api/generate -d "{\"model\":\"$m\",\"keep_alive\":0}" > /dev/null
      echo "    ✓ $m descargado"
    done
    echo "→ Deteniendo servicio Ollama..."
    sudo systemctl stop ollama 2>/dev/null && echo "  ✓ Ollama detenido"
else
    echo "→ Servicio Ollama ya estaba inactivo."
fi

# 4. Liberar la caché del sistema de forma agresiva
echo "→ Liberando page cache y memoria unificada (Drop Caches)..."
sudo sync && sudo sh -c 'echo 3 > /proc/sys/vm/drop_caches'
echo "  ✓ Cache liberada."

# Esperar para estabilización
echo "→ Esperando 5 segundos para estabilización del sistema..."
sleep 5

echo ""
echo "╔══════════════════════════════════════════════════════════════════════════╗"
echo "║                     ESTADO FINAL (REPORTE)                               ║"
echo "╚══════════════════════════════════════════════════════════════════════════╝"
mostrar_diagnostico
echo "════════════════════════════════════════════════════════════════════════════"
EOF
```

Dar permisos de ejecución
Para que el sistema te deje ejecutar el script como un programa, dale permisos con el siguiente comando:

Bash
chmod +x ~/scripts/jetson-clean.sh
Paso 4: Crear el alias permanente (jetson-clean)
Para que no tengas que escribir toda la ruta ~/scripts/jetson-clean.sh cada vez, vamos a añadir el alias a tu configuración de Bash.

Ejecuta este comando para añadir el alias al final de tu archivo .bashrc:

Bash
echo "alias jetson-clean='~/scripts/jetson-clean.sh'" >> ~/.bashrc
Recarga la configuración de tu terminal para que reconozca el nuevo comando inmediatamente:

Bash
source ~/.bashrc
¡Listo! ¿Cómo probarlo?
A partir de ahora, estés en la carpeta que estés, solo debes escribir en tu terminal:

Bash
jetson-clean
Nota: Como el script limpia la caché del sistema (drop_caches), te pedirá tu contraseña de sudo cuando llegue a esa línea.

### ¿Por qué este cambio solucionará tu problema?

* **Filtro Inteligente (`grep -E`):** Ahora buscará palabras clave en tus contenedores. Al encontrar `vllm` y `qwen35-35b`, los meterá en la lista negra y los detendrá de golpe.
* **Caché Real:** Al apagar `qwen35-35b`, los 36.57 GiB se sumarán inmediatamente a tu "Memoria disponible", pasando de esos tristes 14GiB/16GiB a más de 50GiB libres.

Corre el comando `jetson-clean` de nuevo y verás cómo recuperas tu Orin por completo.
